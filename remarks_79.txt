Traceabilty matrix - таблица, где отображается степень покрытия требования тест-кейсами. 
::::
Регресс делаем после каждого изменения (после смоука - основной функционал, критикал пасс, риски), после фикса, перед релизом.
Критикал пасс - сценарий, по которому чаще всего действует пользователь. 
::::
Тестирование рисков - тестирование функционала, который важный для работы, может вызывать риски (пользовательский, финансовые, юридические, технические). 
Типа показ предупреждений, пользовательское соглашение, оферта, соглашение куки. 
::::
Интеграционное тестирование - 
- снизу вверх - функционал небольшими юнитами, потом совместную работу этих юнитов. Меньше багов
- сверху вниз - все модули объединенные сразу. 
по ISTQB
- модульное - модули между собой
- системное - систему в целом, плюс может какие то важные внешние системы.
::::
UI - тестирование графического отображения, соответствие макету. 
::::
Тестирование поля для ввода
- количество символов (классы эквивалетности, граничные значения)
- виды символов
- пустое
- невалидные символы (не те знаки, спецсимволы)
- набор обязательный, как соблюдается
- если пароль - скрытие
- каким методом отправляются введенные данные 
::::
Тестирование конфигурации -
работа системы в различных конфигурациях (ОС версии, память, видеокарты, драйверы, локали), разные гаджеты.
Результат - минимальные требования, рекомендуемые 
::::
Тест-кейсы к БД.
Запросы к базе в sql через IDE
- доступность базы
- наличие таблица
- как вводятся данные
- как сохзраняются данные
- как выдаются данные
- типы данных
::::
Тест-кейсы сложно писать на функционал, который быстро и часто меняется и без документации. 
Либо объемные прекондишены: финансовые транзакции.
::::
Test entry - чем раньше, тем лучше

Exit criteria - критерий окончания тестирования. По эвристике - 
- заказчик доволен
- перестали находиться баги
- функционал покрыт тестами
- много критических багов
::::
Fuzz test - только негативное тестирование. техника тестирования программного обеспечения, 
часто автоматическая или полуавтоматическая, заключающаяся в передаче приложению на вход неправильных, 
неожиданных или случайных данных
::::
